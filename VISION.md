# Web Scraper — Vision

## Проблема
Заказчики на фрилансе чаще всего просят парсить:
- Товары с маркетплейсов (цены, названия, рейтинги)
- Списки компаний / контактов
- Данные для анализа рынка

## Идея проекта
**Парсер товаров с Books to Scrape** (`books.toscrape.com`) — легальный тренировочный сайт, специально созданный для скрапинга. Никаких проблем с TOS.

## Что умеет

| Фича | Зачем в портфолио |
|-------|-------------------|
| Парсинг нескольких страниц (пагинация) | Показывает что умеешь ходить по страницам |
| Retry-логика с exponential backoff | Заказчики это любят — надёжность |
| Rate limiting (задержка между запросами) | Показывает уважение к серверу |
| Экспорт в CSV + JSON | Два формата — гибкость |
| Логирование | Профессиональный подход |
| CLI-интерфейс (argparse) | Удобство запуска с параметрами |
| Сводная статистика после парсинга | Показывает аналитический подход |

## Что парсим
С каждой книги:
- Название
- Цена
- Рейтинг (1-5 звёзд)
- Наличие (in stock / out of stock)
- Категория
- URL

## Структура проекта

```
web-scraper/
├── scraper/
│   ├── __init__.py
│   ├── main.py           # CLI entry point (argparse)
│   ├── parser.py          # Логика парсинга HTML
│   ├── fetcher.py         # HTTP-запросы + retry + rate limit
│   └── exporter.py        # CSV / JSON экспорт
├── output/                # Сюда падают результаты
│   └── .gitkeep
├── requirements.txt
├── .gitignore
└── README.md
```

## Пример использования

```bash
# Парсить первые 5 страниц, сохранить в CSV
python -m scraper.main --pages 5 --format csv

# Парсить всё, сохранить в JSON
python -m scraper.main --all --format json

# Парсить конкретную категорию
python -m scraper.main --category "science" --format csv
```

## Почему это хорошо для портфолио
- Легальный сайт — можно показать заказчику без стыда
- Покрывает 90% того, что просят на фрилансе (пагинация, retry, экспорт)
- Чистый CLI — заказчик видит инструмент, а не просто скрипт
- Код разделён на модули — показывает архитектурное мышление
